

1.  在keil中，定义结构体变量时，出现expected expression的错误提示   keil v5.39.0.0  stm32c011
分析1： 在switch下面的case中定义
分析2： 在stm32cubeide中应不会出现（待验证），在keil中出现
解决： 把结构体的定义放在所有case的上面定义

      typedef struct
      {
          uint8_t tx_len;
          uint8_t rx_len; 
      } auto_mode_spi_trx;

      switch (command)
      {
         //auto_mode_spi_trx   *spi_command_info;      //ok
         case OMMOCOMM_AUTO_CMD_SPI_TXRX:           
              auto_mode_spi_trx   *spi_command_info;    //err  //expected expression
      }

2. keil中编译出现提示： keil v5.39.0.0    stm32c011
     warning: passing arguments to a function without a prototype is deprecated in all versions of C and is not supported in C23 [-Wdeprecated-non-prototype]
  335 |   usart_rx_callback(usart1_rx_buffer, usart1_rx_buffer_len);
分析： 主要是函数指针定义时，没有指向函数 usart_rx_callback_t usart_rx_callback = NULL;
解决： 打开魔法棒，点击C/C++,在Mis Controls中加入下面的一句，重新编译，即ok
          -Wno-deprecated-non-prototype

5.  问题： 提示不能强制转换为 uint32   （nrf52840）
       converting a packed 'device_info_len_t' pointer (alignment 1) to a 'const uint32_t' {aka 'const long unsigned int'} 
       pointer (alignment 4) may result in an unaligned pointer value [-Werror=address-of-packed-member]

    分析： 相关代码如下
         typedef __PACKED_STRUCT // 16 bytes total
         {
             uint16_t perm_len;       /* Device info permanent length */
             uint16_t man_len;        /* Device info manufacturer length */
             uint16_t user_len;       /* Device info user length */
             uint8_t  future_use[10]; /* Reserved for future use */
         } device_info_len_t;

         device_info_len_t empty_device_info __attribute__((aligned(4)));
         memset(&empty_device_info, 0x00, sizeof(device_info_len_t));
         nrf_nvmc_write_words(active_page_addr + sizeof(magic_data_t), (const uint32_t*)&empty_device_info, sizeof(device_info_len_t) >> 2);   //错误提示行

    解决： 在代码的前后加 #pragma，屏蔽编译错误 （意思是这里不需要编译器检查错误）

         #pragma GCC diagnostic push
         #pragma GCC diagnostic ignored "-Waddress-of-packed-member"

                 // write device info len 0
                 device_info_len_t empty_device_info __attribute__((aligned(4)));
                 memset(&empty_device_info, 0x00, sizeof(device_info_len_t));
                 nrf_nvmc_write_words(active_page_addr + sizeof(magic_data_t), (const uint32_t*)&empty_device_info, sizeof(device_info_len_t) >> 2);

         #pragma GCC diagnostic pop


6. 问题： unknown option after '#pragma GCC diagnostic' kind [-Werror=pragmas]
#pragma GCC diagnostic ignored "-Wanalyzer-infinite-loop"  //nrf52840

    分析：#pragma GCC diagnostic ignored "-Wanalyzer-infinite-loop"

    解决1： 去掉GCC, 改成以下
            #pragma diagnostic ignored "-Wanalyzer-infinite-loop"

    解决2：  魔法棒中，右键单击 project \options \ 左上角的下拉选Common \code \compiler \Additional C/C++ Compiler Options
                 填入值： -Wno-pragmas

8.  context的作用 （待举例说明来完善）

    俺打一个比喻，context就是，用一个快递盒子把用户的东东装在里面，在盒子上贴一个独一无二的用户标签，该标签就是context
    像一个模块函数被多个地方调用，用context可以区分不同的调用地方

    打个比喻：
      comms_1wire  -- 小孩
      context=this    -- 小孩名字    
      doghut            --  小孩的狗咬人
      this->findfather  -- 通过小孩找到父亲索赔
